Written documentation stating the concept,user interactions, purpose etc.


The concept for this project is simple: we had decided to make a mock universe through JavaScript as a way of exploring visual interaction. The base idea was to have a couple of planets circling a central sun and a couple of other celestial bodies present that could be interacted with. Of final product has three fictional planets circling a sun with their satellites, which the user can influence the speed of; stars and dust clouds quivering; and asteroids that can be provoked on click. Our aim was to explore the idea of cause and effect to bring to the forefront the impact we have on the world around us, especially as discussions surrounding interstellar colonies become less science-fiction and more reality. The project creates classes for the various elements, using inheritance as a way of ensuring that some of the objects work the same way. The planets and satellites, for example, are two objects (the latter inheriting from the former); the only difference being that the planets’ center is the sun and the satellites’ is a given planet, making it necessary for them to have an additional function that updates their center’s position as it is constantly evolving.  



Written documentation stating the process: how the project evolved and what design choices were made and why.


The process for this project started with us conceptualizing what kind of environment we wanted to create, which was clear from the get go as Doug came with the idea to create a universe and we agreed on it. We broke down the elements that we wanted in our universe to planets, sun, asteroids, satellites, stars and dust clouds, with each performing different movements and behaving differently.

We began the project using Three.js, however this library proved to be overly complicated, so we instead decided to use the p5.js library as it was more similar to Processing which we had both used before. We also started the project off in 3D space, however this proved to be complicated and tedious so we changed to 2D, as it would still give us the effect we were going for.

We also had to reign in some of the ideas that we had for the project, for example implementing black holes that would somehow suck up all other on screen elements. We also originally wanted the mouse interaction with the planets to be able to push them out of orbit completely and to get rid of the planets centre of gravity, however this also proved to be complicated.

We looked at various p5 features such a play elements for the creation of sprites and also the tickle feature to try and apply and hack these into some of our objects however we had trouble applying these. Finally, a key design choice that was made was to give all of the objects abstract and evocative overlaid textures, particularly the planets and the sun. We also wanted to give all of the objects movement, and to also make as many objects move and interact with the users mouse as possible.

For the workload, we have “separated” it into three basic parts: core implementation, additional coding and visual elements. While Dougy took it care of the core implementation of the basic element – basic classes like celestial bodies, planets and satellites, as well as the orbiting logic – Imogen took care of all the visual components of the project as well as the coding of some elements such as the dust clouds & stars noise movements as well as the asteroids spawning through mouse clicks.
